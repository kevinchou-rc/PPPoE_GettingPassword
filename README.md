# 🚀 PPPoE Server Simulator (PAP Credential Sniffer)

这是一个基于 Python 和 Scapy 编写的轻量级 PPPoE 服务端模拟器。它通过与真实路由器进行底层数据链路层（Layer 2）交互，模拟完整的宽带发现与会话过程，并在 LCP 阶段强制降级为 PAP 认证，从而截获并明文显示宽带拨号的账号与密码。

> **⚠️ 安全免责声明**
> 本工具仅供**网络协议学习、合规的安全研究与授权测试**使用。请勿用于任何非法的网络监听或未经授权的网络环境。使用者需自行承担因不当使用而引发的任何法律后果与责任。

---

## ✨ 核心特性

- **无缝底层交互**：不依赖操作系统的上层 Socket，直接读写数据链路层的 `0x8863` (Discovery 阶段) 和 `0x8864` (Session 阶段) 以太网帧。
- **强制协议降级 (Downgrade Attack)**：通过拦截并修改 LCP 协商报文，迫使客户端放弃加密的 CHAP 协议，改用明文的 PAP 协议。
- **自动化凭据提取**：自动解构拦截到的 PAP 数据包，将字节流解析为清晰可读的明文账号和密码。

## ⚙️ 环境依赖

运行本脚本需要满足以下条件：

1. **Python 环境**：Python 3.6 或更高版本。

2. **核心依赖库**：`scapy`

   ```bash
   pip install scapy
   ```


1. **系统底层抓包支持**：
   - **Windows**: 必须安装 [Npcap](https://npcap.com/) (通常随 Wireshark 一起安装)。安装时请勾选“支持 raw 802.11 流量”及兼容 WinPcap API 选项。
   - **Linux / macOS**: 系统原生支持，但执行脚本时**必须使用 `root` 权限**。

## 🚀 快速上手

### 1. 修改配置

使用文本编辑器打开 Python 脚本文件，根据实际情况修改顶部的配置区：

```python
# ================= 配置区 =================
# 填入你要监听的物理网卡名称。
# Windows 示例: "Ethernet" 或 "WLAN"
# Linux 示例: "eth0", "enp3s0"
# 设为 None 时，Scapy 会尝试使用默认网卡
INTERFACE = None  

# 伪造的宽带服务器名称 (AC-Name)
AC_NAME = b"Python-vBNG-Server"
# ==========================================
```

### 2. 连接硬件

找一根网线，将待测试路由器（或拨号设备）的 **WAN 口**直接连接到运行此脚本电脑的**网口**上。

### 3. 运行服务

打开命令行终端，**以最高权限**运行脚本：

Bash

```bash
# Windows (以管理员身份运行 CMD 或 PowerShell)
python pppoe_sniffer.py

# Linux / macOS
sudo python3 pppoe_sniffer.py
```

*此时，终端应提示正在监听网络接口，等待拨号。*

### 4. 发起拨号

进入待测路由器的 Web 管理后台：

1. 找到“网络设置”或“WAN口设置”。
2. 将连接类型更改为 **PPPoE（宽带拨号）**。
3. 输入任意测试账号和密码，点击“连接”或“保存”触发拨号请求。

### 5. 获取凭据

观察脚本终端输出，当底层的四步握手和 LCP 协商完成后，脚本会自动拦截并打印出明文账号密码：

Plaintext

```bash
==================================================
!!! 成功拦截 PAP 认证数据包 !!!
==================================================
[*] 宽带账号 (Username) : my_test_account
[*] 宽带密码 (Password) : super_secret_password
==================================================
```
